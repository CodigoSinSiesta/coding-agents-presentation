# Cómo Evitar Alucinaciones

## Tiempo estimado
4 minutos

## Puntos clave
- **Técnicas de prevención** (Do's):
  - **Contexto explícito**: Da al agente la información que necesita
  - **Instrucciones específicas**: Sé específico en el System Prompt. Ambigüedad = alucinaciones
  - **Pide verificación**: Si no estás seguro, pregunta antes de inventar
  - **Revisa y verifica**: El código generado no es tuyo hasta que lo entiendes
- **Técnicas de corrección** (Don'ts):
  - No copies APIs o paquetes sin verificar que existen
  - No confíes ciegamente en que el agente "sabe" lo que hace
  - No dejes que tareas enormes en un solo prompt
  - No uses el agente como sustituto de entender el problema
- **Flujo de trabajo seguro**:
  1. Explorar → 2. Planificar → 3. Implementar → 4. Verificar
- La regla de oro: SIEMPRE VERIFICA antes de confiar

## Datos relevantes
- Claude Code puede validar imports automáticamente con el lenguaje
- Una pregunta de ahorra tokens y tiempo
- La revisión crítica debe encontrar bugs que el agente nunca detectaría

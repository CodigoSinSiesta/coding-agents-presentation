# Buenas Prácticas (Do's y Don'ts)

## Tiempo estimado
4 minutos

## Puntos clave
- **DO's** (Haz siempre):
  - Usa el agente para tareas repetitivas, boilerplate
  - Dale contexto explícito: arquitectura, tests, documentación
  - Haz preguntas específicas, no vagas
  - Revisa SIEMPRE el código generado
  - Itera: el agente mejora con feedback
  - Empieza pequeño, crece gradualmente
  - Usa .cursorrules / CLAUDE.md
- **DON'Ts** (Nunca hagas):
  - No copies APIs o paquetes sin verificar
  - No confíes ciegamente en lo que el agente dice
  - No dejes que tareas enormes en un solo prompt
  - No uses el agente como sustituto de entender el problema
  - No olvides que el agente no sabe lo que no le dices

## Datos relevantes
- El agente es JARVIS, tú eres Tony Stark
- Tú diriges, el agente ejecuta
- El contexto es poder: más contexto = mejores resultados
- La revisión es obligatoria: el código no es tuyo hasta que lo entiendes
